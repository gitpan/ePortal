NAME

faq-rus - ePortal's FAQ in Russian

Что это такое?

ePortal - WEB сервер предприятия. Предназначен для использования во внутренней сети (Intranet). Позволяет решить часть проблем, связанных с обеспечением единого справочного пространства в ЛВС, организовать хранение персональной информации пользователей сети и т.д.

Какие модули входят в состав ePortal?

В настоящее время в состав ePortal входят следующие модули:

  • Система проверки прав доступа (ACL).
    Для большей части объектов ePortal применимо понятие ``прав доступа''. Например, можно определить персонально пользователя или группу пользователей, имеющих различные уровни доступа к какому-то об'екту.

  • Каталог ресурсов.
    Если Вы видели Yahoo, то представляете что такое Каталог. В ePortal реализованы функции создания групп и ссылок в каталоге. Каждая ссылка может быть либо ссылкой на ресурс, либо файлом, либо текстом. Можно производить поиск по каталогу.

  • Домашная страница портала.
    Домашняя страница состоит из разделов. Есть также список доступных разделов, который конкретизируются администратором. Администратор формирует домашнуюю страницу сервера, а любой зарегистрированный пользователь может сформировать персональную домашнуюю страницу по своему вкусу.

  • Дискуссионные форумы.
    Здесь можно что-либо обсудить. Можно также организовать ``Новости сервера'' с возможность обсуждения и т.д. Право создания сообщения, ответа, просмотра регулируется с помощью ACL.

  • Просмотр содержимого файловой системы
    Это доступ к обычным папкам и файлам, но оформление папок производится не выпадая из общего стиля сайта.

    Можно регулировать содержание списка файлов, задавать описание файлов (в т.ч. и с помощью функции), описание для стандартных файлов формируется автоматически.

    Если файлы сложены в каталоги YYYY/MM/DD, то можно вывести с краю экрана календарь и быстро перемещаться между датами.

  • Пользователи ePortal
    Пользователей может регистраировать администратор вручную, либо можно задействовать ``внешнюю'' процедуру регистрации. В данный момент работает процедура регистрации пользователей на основе LDAP сервера Novell Netware. В дальнейших планах подключить PAM.

  • Приложения.
    Функционально независимые модули реализованы в виде Приложений. Каждое приложение распространяется, устанавливается и конфигурируется независимо друг от друга. Подробный список готовых приложений и их возможности см.ниже.

Приложения ePortal

Organizer - Персональный Органайзер

Персональный органайзер автоматически создается для каждого зарегистраированного пользователя. Могут быть также общие органайзеры с регулируемым доступом.

  • Записная книжка
  • Список дел
    Текущие дела, напоминание, выполненные дела и т.д.

  • Календарь
    Классический ежедневник с получасовым интервалом для заданий. Уведомление посредством всплывающего окна о наступлении события (JavaScript). Надо лишь, чтобы browser находился на любой странице ePortal.

  • Адреса
    Адресная книжка. Все как обычно.

  • Даты и годовщины
    Напоминание о датах и годовщинах. Включен большой список стандартных дат (праздники, дни рождения знаменитостей, именины и т.д.)

OffPhones - Телефонный справочник

Телефонный справочник предприятия. Поддерживается иерархическая структура подразделений, несколько видов телефонных номеров (внутренный, ГТС, факс и т.д.).

Зарегистрированные пользователи могут делать свои личные пометки в справочнике.

О замеченных неточностях можно написать администратору.

MsgForum - Дискуссионные форумы

Обсуждение вопросов, режим ``Вопрос-ответ'', ленты новостей и т.д. Поддерживаются красивые ``смайлики'', вставка кода HTML и т.д.

Diskoteka - Каталог носителей информации

В Дискотеке хранится оглавление носителей информации: дискеты, CD диски и т.д. К каждому файлу или каталогу можно добавить описание. Программные продукты могут размещаться на нескольких ностелях. Поиск...

Сколько это стоит?

На момент написания этой документации ePortal является Open Source Software, что означает, что вы вправе свободно распространять его (копировать, передавать) в виде исходных текстов при условии, что все ссылки на авторство останутся без изменения.

Я за собой оставляю право в любой момент изменить условия распространения и использования ePortal.

В любом случае прошу всех, кто использует ePortal в своей деятельности или в деятельности своего предприятия выслать мне краткую анкету:

 - ФИО для контакта.
 - email.
 - Название предприятия.
 - WWW сервер предприятия (если есть).
 - Профиль деятельности предприятия.
 - Кол-во пользователей ePortal (если это не ДСП).

Со своей стороны обещаю всем выславшим анкету всяческое содействие, а именно:

 - Уведомление о выходе новых версий.
 - Приоритетная реализация просьб от зарегистрированных пользователей.
 - Помощь по e-mail по вопросу установки, настройки и использования ePortal.

Если кто-то имеет возможность и желание поддержать этот проект финансово, то я буду только рад. Все вопросы прошу согласовывать по email: rusakov_sa@users.sourceforge.net.

Установка ePortal

Что для этого необходимо?

Для работы ePortal необходимо следующее:

 - Apache WEB server
 - MySQL database server
 - mod_perl for Apache
 - Perl 5.6.1
 - HTML::Mason perl package

ePortal тестировался на следующих платформах:

 - Linux RedHat 7.3
 - Windows 2000 Prof

Крайне не рекумендуется устанавливать ePortal на Windows 9x. Работать будет, но скорость вас сильно разочарует.

Для Windows платформы я брал уже скомпилированный Apache WEB server и Perl от ActiveState. MySQL тоже замечательно работает под NT.

Некоторые perl packages недоступны на сайте ActiveState. Посмотрите здесь: http://theoryx5.uwinnipeg.ca/

Последовательность установки

  • Установить и настроить Apache, MySQL, Perl, mod_per, HTML::Mason. Необходимо убедиться, что все это работает (пока без ePortal).

  • По умолчанию ePortal устанавливается в каталог /home/ePortal или c:\home\ePortal (для Windows) 
     perl Makefile.PL  HOME=/home/ePortal
 make
 make install

    Этим вы установите Perl модули ядра ePortal и скопируете все остальные файлы в дерево каталогов, на которое указывает HOME. По умолчанию HOME=/home/ePortal.

  • Каждое приложения, которые вы устанавливаете, распакуйте в отдельный каталог и аналогично: 
     perl Makefile.PL  HOME=/home/ePortal
 make
 make install

  • Теперь надо создать базу данных MySQL 
     mysqladmin create eportal

  • Настройка ePortal производится через WEB интерфейс, но прежде чем его запустить в первый раз необходимо указать, где находится база дыннх MySQL.

    Для этого необходимо задать переменные среды окружения. Для Юникс это будет так:

     export EPORTAL_DBI_SOURCE=dbi:mysql:host=mysql.domain;database=ePortal;
 export EPORTAL_DBI_USERNAME=root
 export EPORTAL_DBI_PASSWORD=******

    Естественно, что имя пользователя, базы и пароль у Вас могут быть другими.

    Для Windows NT переменные среды задаются через ``Панель управления'', ``Система''.

  • Создаем первоначальную структуру таблиц в базе данных: 
     cd bin
 perl ePortal.pl

    Эта команда проверит и при необходимости создаст все таблицы на сервере БД. Эту команду можно выполнять любое количество раз. Если таблицы уже существуют, то они будут сконвертированы к текущей версии без потери данных.

  • Настраиваем Apache. Лучше всего взять за образец конфигурационный файл 
     samples/httpd.conf

    и внимательно его прочитать. Все ключевые моменты там задокументированы.

  • Для первого запуска надо установить значение параметра admin_mode в 1. 
     httpd.conf:
 PerlSetVar ePortal_admin_mode 1

    В этом режиме любой, кто обратится к сайту автоматически наделяется правами администратора. Этот режим необходим на время первоначального запуска и конфигурирования ePortal, регистрации настоящих пользователей. Затем его следует выключить.

  • Теперь можно запускать Apache и первым делом обратиться по адресу http://localhost/admin/index.htm.

    Здесь вам надо зарегистрировать пользователей ePortal и определить список доступных разделов для домашней страницы.

  • Необходимо обеспечить периодический запуск скриптов из каталога bin/cron.d - каждые 5 минут, bin/cron.hourly - каждый час, bin/cron.daily - каждый день по ночам.

    Для пользователей Unix прилагается пример файла crontab: samples/crontab

  • Если вы установили режим вывода отладочных сообщений в файл (параметр log /var/log/ePortal.log), то для автоматической смены лога можно использовать прилагаемый конфигурационный файл для logrotate (samples/logrotate.d/ePortal).

    Для пользователей WinNT это придется делать своими средствами, либо просто руками периодически удалять этот файл.

  • Для увеличения производительности рекомендуется установить переменную окружения: 
     учзщке PERL_NO_VALIDATION=1

    Для подробностей смотрите документацию Params::Validate.

  • Теперь можно создать домашнюю страницу по умолчанию, ввести ссылки в меню...

    Пожалуй что все...
